## 작업 흐름을 돌아보며

### 1. NumPy 배열을 정렬하고 다시 배치했을 때

README 이후의 NumPy 학습은 배열을 계산하는 것에서 배열의 순서와 구조를 바꾸는 쪽으로 이어졌다.

먼저 `np.sort()`로 배열을 정렬하고, `axis=None`, `axis=0`, `axis=1`에 따라 결과가 어떻게 달라지는지 확인했다. 전체 값을 하나의 1차원 배열처럼 정렬할 때와, 열 또는 행 방향으로 정렬할 때의 결과가 다르다는 점이 중요했다.

정렬 자체보다 더 의미 있었던 부분은 `np.argsort()`였다. 값이 정렬된 결과만 보는 것이 아니라, 정렬된 순서에 해당하는 인덱스를 얻고 그 인덱스로 이름, 점수, 작업 우선순위 같은 다른 배열을 함께 재배치할 수 있었다.

- `np.sort()`와 원본 배열 유지 여부
- 슬라이싱을 이용한 내림차순 정렬
- `axis` 기준에 따른 2차원 배열 정렬
- `np.argsort()`로 정렬 순서 인덱스 얻기
- 점수와 이름 배열을 같은 순서로 정렬하기

관련 기록: [`05_sort.ipynb`](01_numpy/05_sort.ipynb)

### 2. 배열을 붙이고 나누면서 shape 조건을 다시 확인했을 때

정렬 다음에는 배열의 삽입, 삭제, 병합, 분리를 다뤘다.

`np.insert()`와 `np.delete()`는 원본 배열을 직접 바꾸는 것이 아니라 결과가 반영된 새 배열을 반환한다는 점을 확인했다. 특히 2차원 배열에서 `axis`를 지정하지 않으면 배열이 1차원처럼 처리되고, `axis=0`과 `axis=1`에서는 행과 열 기준으로 동작이 달라졌다.

병합에서는 shape 조건이 다시 중요해졌다. `append`, `vstack`, `hstack`, `concatenate`는 모두 배열을 붙이는 함수지만, 어느 축으로 붙이는지에 따라 필요한 shape 조건이 달랐다. 앞에서 배운 `axis` 개념이 여기서 다시 연결됐다.

- `np.insert()`로 지정 위치에 값 삽입하기
- `np.delete()`로 특정 위치의 값 제거하기
- `np.append()`와 원본 배열의 관계 확인하기
- `np.vstack()`, `np.hstack()`으로 행과 열 방향 병합하기
- `np.concatenate()`에서 `axis` 기준으로 병합하기
- `split` 계열 함수로 배열 분리하기

관련 기록: [`06_merge_split.ipynb`](01_numpy/06_merge_split.ipynb)

### 3. reshape에서 차원과 원본 공유를 의식하게 됐을 때

배열 형태 변경에서는 단순히 모양을 바꾸는 것보다, 원소 개수와 메모리 공유를 함께 생각해야 했다.

`reshape()`는 전체 원소 개수가 맞아야 원하는 shape로 바꿀 수 있었다. `-1`을 사용하면 NumPy가 남은 차원의 크기를 자동 계산해주지만, 이것도 원소 개수 조건이 맞을 때만 가능했다.

`ravel()`과 `flatten()`의 차이도 중요한 지점이었다. 둘 다 다차원 배열을 1차원으로 만들지만, `ravel()`은 원본과 값을 공유할 수 있고 `flatten()`은 복사본을 만든다는 점에서 이후 수정 결과가 달라진다.

- `reshape()`로 차원과 형태 변경하기
- `-1`을 이용해 남은 차원 자동 계산하기
- `ravel()`과 `flatten()`의 차이 확인하기
- `expand_dims()`로 차원 추가하기
- `squeeze()`로 길이가 1인 축 제거하기
- `transpose()`로 행과 열 바꾸기

관련 기록: [`07_reshape.ipynb`](01_numpy/07_reshape.ipynb)

### 4. Pandas로 넘어가며 행과 열 중심으로 생각하기 시작했을 때

NumPy에서는 배열의 shape와 축을 기준으로 생각했다면, Pandas에서는 인덱스, 컬럼, 행 단위의 레코드를 함께 봐야 했다.

처음에는 `Series`를 통해 값과 인덱스가 함께 움직이는 구조를 확인했다. 기본 숫자 인덱스뿐 아니라 직접 지정한 라벨 인덱스, 날짜 인덱스를 사용하면서 `loc`와 `iloc`의 차이도 같이 연습했다.

이후 `DataFrame`에서는 딕셔너리, 리스트, 튜플 등 여러 자료구조에서 표 형태 데이터를 만드는 방법을 익혔다. `head`, `tail`, `info`, `describe`, `value_counts` 같은 메서드는 데이터를 본격적으로 처리하기 전에 구조와 분포를 확인하는 도구로 사용했다.

- `Series` 생성과 인덱스 지정
- `loc`, `iloc` 기반 데이터 접근
- 날짜 인덱스와 정규분포 샘플 데이터 생성
- 딕셔너리, 리스트, 튜플을 이용한 `DataFrame` 생성
- 컬럼 추가와 삭제, 행 추가와 삭제
- 조건식으로 필요한 행 필터링하기
- 정렬과 기본 메서드로 데이터 확인하기

관련 기록: [`01_series.ipynb`](02_pandas/01_series.ipynb), [`02_dataframe.ipynb`](02_pandas/02_dataframe.ipynb)

### 5. 전처리에서 데이터가 깔끔하지 않다는 점을 직접 다뤘을 때

Pandas 전처리에서는 실제 데이터가 항상 분석하기 좋은 형태로 들어오지 않는다는 점을 확인했다.

연락처 데이터와 Titanic 데이터를 사용하면서 중복값, 결측치, 이상치, 문자열 처리, 타입 변환을 다뤘다. 특히 중복 제거는 단순히 같은 이름을 지우는 문제가 아니라, 어떤 컬럼 조합을 기준으로 같은 데이터라고 판단할지 정해야 했다.

결측치도 마찬가지였다. 무조건 삭제할지, 평균이나 중앙값으로 채울지, 특정 의미를 가진 값으로 대체할지는 데이터의 성격에 따라 달라졌다. 이 과정에서 전처리는 함수 사용법보다 기준을 세우는 일이 더 중요하다는 것을 알게 됐다.

- `duplicated()`와 `drop_duplicates()`로 중복 확인 및 제거
- `isna()`, `notna()`로 결측치 확인
- `fillna()`, `dropna()`로 결측치 처리
- 컬럼 타입 변환과 날짜 데이터 처리
- 문자열 컬럼 정리
- 조건식을 이용한 이상치 탐색

관련 기록: [`03_preprocessing.ipynb`](02_pandas/03_preprocessing.ipynb)

### 6. 집계와 피벗테이블로 데이터를 요약했을 때

집계 학습에서는 개별 행을 보는 것에서 그룹 단위로 데이터를 요약하는 방식으로 넘어갔다.

`groupby()`를 사용하면서 부서별, 직급별, 구간별로 데이터를 묶고 평균, 합계, 개수, 표준편차 같은 값을 계산했다. 앞에서 통계 시간에 정리했던 평균과 표준편차가 Pandas 집계 함수로 다시 등장했다.

`agg()`는 여러 집계 함수를 한 번에 적용할 수 있어 단순한 평균 계산보다 더 넓은 요약표를 만들 수 있었다. `filter()`는 그룹별 조건을 기준으로 행을 남기는 방식이라, 개별 행 필터링과는 다른 관점이 필요했다.

Titanic 과제에서는 출항지별 승객 수와 생존자 수, 나이 구간별 생존률을 구하면서 `pd.cut()`과 `pivot_table()`을 함께 사용했다.

- `groupby()`로 그룹별 집계하기
- `agg()`로 여러 집계 함수 적용하기
- 그룹 기준 `filter()` 사용하기
- 급여 구간처럼 새로운 분류 컬럼 만들기
- `pivot_table()`로 요약표 만들기
- Titanic 데이터로 생존률 집계하기

관련 기록: [`04_aggregation.ipynb`](02_pandas/04_aggregation.ipynb), [`homework_titanic_agg.ipynb`](02_pandas/homework/homework_titanic_agg.ipynb)

### 7. 여러 DataFrame을 합치며 관계를 생각했을 때

병합 학습에서는 여러 표를 하나의 분석 가능한 형태로 합치는 방법을 다뤘다.

`merge()`에서는 inner, left, right, outer join의 차이를 확인했다. 공통 키가 있는 행만 남길지, 왼쪽 또는 오른쪽 데이터를 기준으로 유지할지에 따라 결과 행 수와 결측치가 달라졌다.

`join()`은 인덱스 기반 결합을 이해하는 데 도움이 됐고, `concat()`은 같은 구조의 데이터를 위아래 또는 좌우로 이어 붙일 때 사용했다. 고객, 주문, 상품 데이터 예제에서는 단순히 표를 붙이는 것이 아니라 1:N 관계를 따라 필요한 정보를 연결해야 했다.

- `merge()`로 공통 키 기준 병합하기
- inner, left, right, outer join 결과 비교하기
- `join()`으로 인덱스 기반 결합하기
- `concat()`으로 행 또는 열 방향 연결하기
- `reset_index()`로 인덱스 정리하기
- 고객, 주문, 상품 데이터를 하나의 조회 결과로 만들기

관련 기록: [`05_merge_join.ipynb`](02_pandas/05_merge_join.ipynb)

## 아쉬웠던 점

학습 속도가 빨라지면서 노트북 안에는 당시의 이해와 실수가 그대로 남아 있다.

- `axis`, `reshape`, `ravel`, `broadcasting` 같은 단어에 오타가 일부 남아 있다.
- 주석에는 이해한 말을 바로 적은 흔적이 많지만, 표현이 아직 정리되지 않은 부분이 있다.
- 일부 예제는 앞 셀에서 만든 변수에 의존하므로 중간부터 실행하면 오류가 날 수 있다.
- 전처리 기준은 코드로는 남아 있지만, 왜 그 기준을 선택했는지 설명이 부족한 부분이 있다.
- Pandas 병합 예제는 join 종류별 결과 차이를 표로 비교하면 더 이해하기 쉬울 것 같다.
- 과제 파일은 문제 풀이 흔적은 있지만, 풀이 과정에 대한 설명이 더 보완되면 복습 자료로 쓰기 좋다.

이 아쉬운 점들은 정리되지 않은 부분이기도 하지만, 동시에 공부가 진행 중이라는 흔적이기도 하다. 다만 나중에 다시 볼 자료로 만들려면 오타, 셀 실행 순서, 설명 부족은 반드시 보완해야 한다.
